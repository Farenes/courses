# Темы курсовых работ
1. Desktop или Web визуализатор алгоритмов сортировки (просто)
	- **Что делать**: Создать десктопное приложение, которое превращает абстрактные алгоритмы сортировки в зрелищное и понятное визуальное представление. Цель — не просто отсортировать массив, а показать _как_ это происходит, шаг за шагом, раскрывая внутреннюю логику и эффективность (или неэффективность) каждого алгоритма.
2. Симулятор распределения памяти (аллокатор) (сложно)
	- **Суть проекта:** Реализация и сравнение различных алгоритмов управления памятью.
	- **Что делать:** Реализовать симуляцию работы heap-памяти с алгоритмами аллокации: First Fit, Best Fit, Buddy System. Сгенерировать последовательность запросов на выделение и освобождение памяти, визуализировать состояние "кучи" и сравнить алгоритмы по степени фрагментации и скорости работы.
3. Реализация системы кэширования с различными политиками вытеснения (средне)
	- **Суть проекта**: Создание эффективной структуры данных для кэша.
	- **Что делать**: Реализовать кэш с интерфейсом get(key) и put(key, value). Реализовать и сравнить политики вытеснения: Least Recently Used (LRU), Least Frequently Used (LFU), FIFO. LRU/LFU должны работать за O(1) на операцию. Написать бенчмарки для сравнения их эффективности на разных паттернах доступа.
4. Реализация Garbage Collector из Java (сложно)
5. Потоковые алгоритмы (Streaming Algorithms) (сложно)
	- **Суть проекта**: Реализация алгоритмов для обработки больших потоков данных в один проход с малым потреблением памяти.
	- **Что делать**: Реализовать и протестировать алгоритмы: Count-Min Sketch для оценки частоты элементов, HyperLogLog для оценки количества уникальных элементов, Reservoir Sampling для случайной выборки. Сгенерировать большой поток данных и показать точность алгоритмов.
6. R-tree для хранения и поиска геопространственных данных (сложно)
	- **Суть проекта**: Реализация структуры данных для эффективного поиска объектов на плоскости (запросы "найти все объекты в этом прямоугольнике").
	- **Что делать**: Реализовать R-tree с алгоритмами вставки, удаления и поиска по диапазону. Визуализировать дерево и процесс поиска для демонстрации. В качестве тестовых данных использовать синтетические точки или реальные координаты городов.
7. Система сжатия файлов на основе Huffman + LZ77 (сложно)
	- **Суть проекта**: Реализация двух популярных алгоритмов сжатия и их сравнение на различных типах данных.
	- **Что делать**: Написать кодер и декодер для алгоритма Хаффмана (статический и адаптивный) и для алгоритма LZ77 (или LZW). Провести тестирование на текстах, картинках (как на бинарных данных), измерить степень сжатия и скорость работы. Сложность — в эффективной работе с битовыми потоками.
8. Моделирование внешней сортировки (сортировка больших файлов) (средне)
	- **Суть проекта**: Реализация алгоритма, который сортирует данные, не помещающиеся в оперативную память.
	- **Что делать**: Реализовать классическую внешнюю сортировку с помощью слияния: разбиение исходного файла на части, сортировка каждой части в памяти (например, quicksort), и многофазное слияние отсортированных частей. Сложность — в эффективном управлении буферами ввода-вывода.
9. Модель клеточного автомата (например, "Жизнь" Конвея) с расширениями (просто)
	- **Что делать**: Реализовать бесконечное поле для игры "Жизнь" с эффективными структурами данных (например, хранить только живые клетки и их соседей). Добавить возможность загрузки паттернов, шаг вперед/назад, измерение производительности. Реализовать другие правила клеточных автоматов.
10. Планер задач (Task Scheduler) с системами приоритетов (сложно)
	- **Что делать**: Реализовать несколько структур данных для планирования задач: очередь с приоритетами на основе кучи, круговая буферизация. Реализовать различные дисциплины обслуживания: FIFO, LIFO, SJF (Shortest Job First), Round Robin. Сравнить их эффективность на различных нагрузках.
11. Tries (Префиксные деревья) и их применение (сложно)
	- **Что делать**: Реализовать несколько видов префиксных деревьев: стандартное trie, сжатое (radix tree), суффиксное дерево (задача со звездочкой). Применить их для решения практических задач: автодополнение, подсчет количества уникальных подстрок, поиск всех вхождений образца в текст (алгоритм Ахо-Корасик).
12. Приложение для стеганографии (средне)
	- **Суть проекта**: Десктоп-программа для скрытия текстового сообщения внутри изображения (формат BMP или PNG) и его последующего извлечения.  
13. Калькулятор матриц с продвинутыми операциями (просто)
	- **Суть проекта**: Не просто калькулятор, а инструмент для визуального исследования линейно-алгебраических операций. Пользователь вводит матрицы и видит результат операций, а также их геометрическую интерпретацию (для 2D/3D).
14. AI Racing Sandbox
	- **Суть проекта**: приложение, которое позволяет добавлять свои алгоритмы для проезда машинок, трасса для симуляции.
15. AI Пошаговые солдатики Sandbox
	- **Суть проекта**: приложение, которое позволяет добавлять свои алгоритмы для проезда машинок, трасса для симуляции.
