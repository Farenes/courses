# Лабораторная №1. Основы работы с Git через CLI: взаимодействие с GitHub

## Цель работы
Освоить базовые операции работы с системой контроля версий Git через командную строку, научиться создавать репозитории, управлять изменениями и взаимодействовать с GitHub.

## Теоретическая часть
Git - распределенная система контроля версий, позволяющая отслеживать изменения в файлах и совместно работать над проектами. GitHub - платформа для хостинга Git-репозиториев.

Основные понятия:
- **Репозиторий** - хранилище кода и истории изменений
- **Коммит** - фиксация изменений с описанием
- **Ветка** - параллельная линия разработки
- **Клон** - локальная копия удаленного репозитория
- **Пуш/пулл** - отправка/получение изменений в удаленный репозиторий

## Предварительные требования
1. Установленный Git ([скачать для Windows](https://git-scm.com/))
2. Аккаунт на GitHub ([регистрация](https://github.com/))
3. Токен доступа GitHub (Settings → Developer settings → Personal access tokens)

## Задание
Выполнить последовательность команд для работы с Git через CLI и предоставить отчет о выполнении.

## Ход работы

### Часть 1. Настройка и создание репозитория
1. Настройка идентификации
```bash
git config --global user.name "Ваше Имя"
git config --global user.email "ваша_почта@example.com"
```

2. Создание локального репозитория
```bash
mkdir git-lab-project
cd git-lab-project
git init
```

3. Создание файлов проекта
```bash
echo "# Мой первый проект на Git" > README.md
mkdir src
echo "print('Hello, Git!')" > src/main.py
```

### Часть 2. Основные операции Git
4. Проверка состояния и добавление файлов
```bash
git status
git add README.md
git add src/
git status
```

5. Создание первого коммита
```bash
git commit -m "Initial commit: добавлены README и основной код"
```

6. Просмотр истории коммитов
```bash
git log --oneline
```

### Часть 3. Работа с ветками
7. Создание и переключение на новую ветку
```bash
git checkout -b feature/add-config
```

8.Внесение изменений в новой ветке
```bash
echo "DEBUG = True" > config.py
git add config.py
git commit -m "Добавлен файл конфигурации"
```

9. Возврат в основную ветку и слияние
```bash
git checkout main
git merge feature/add-config
```

### Часть 4. Работа с GitHub
10. Создание репозитория на GitHub
- Залогиньтесь на GitHub
- Нажмите "+" → "New repository"
- Имя git-lab-project (можно использовать свое)
- Тип: Public
- Не добавляйте README, .gitignore или license

11. Связывание с удаленным репозиторием
```bash
git remote add origin https://github.com/ВАШ_АККАУНТ/git-lab-project.git
git branch -M main
```

12. Отправка кода на GitHub
```bash
git push -u origin main
При запросе логина используйте username и токен доступа
```

### Часть 5. Имитация командной работы
13. Внесение изменений через другой "компьютер" (сделать в другом каталоге компьютера)
```bash
cd ..
mkdir git-lab-clone
cd git-lab-clone
git clone https://github.com/ВАШ_АККАУНТ/git-lab-project.git
cd git-lab-project
```

14. Создание изменений и отправка
```bash
echo "# Дополнительная информация" >> README.md
git add README.md
git commit -m "Обновлен README"
git push
```

15. Получение изменений в первом репозитории
```bash
cd ../../git-lab-project
git pull
```
