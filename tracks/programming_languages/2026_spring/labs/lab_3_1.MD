# Лабораторная №3 (альтернатива).  Разработка плагина для сервера Minecraft

## Тема и цель работы
Тема: Разработка и интеграция серверного плагина для платформы Spigot (Minecraft Java Edition).

Цель работы: Изучить архитектуру серверных модификаций для Minecraft, освоить принципы работы с Spigot API, а также получить практические навыки создания, компиляции и развертывания плагина, расширяющего функционал игрового сервера.

## Общая постановка задачи
Необходимо разработать плагин для Spigot-сервера, который добавляет на сервер новую механику взаимодействия или автоматизации. Плагин должен быть написан на языке Java, корректо обрабатывать события сервера, иметь файл конфигурации и логику сохранения данных между перезапусками сервера.

## Функциональные требования
1. Основная механика: должна быть реализована основная логика согласно задания ниже.
2. Обработка событий: должна быть реализована обработка событий, согласно задания.
3. Команды: плагин должен предоставлять как минимум одну команду.
4. Конфигурация: плагин должен читать настройки из файла `config.yml`.
5. Хранение данных: Плагин должен уметь сохранять какую-либо статистику в файл (YAML или JSON) и загружать её при старте сервера или входе игрока.

## Отчет о выполнении
1. Титульный лист.
2. Введение: Краткое описание решаемой проблемы и идеи плагина.
3. Структура проекта: Схема классов плагина с кратким описанием назначения каждого класса (главный класс, слушатели событий, классы для работы с данными).
4. Листинг кода: распечатка исходного кода основных компонентов плагина с обязательными комментариями к ключевым участкам кода (регистрация команд, обработка событий, работа с конфигом).
5. Примеры работы: Скриншоты, демонстрирующие работу плагина на сервере (выполнение команд, вывод сообщений в чат, изменение игрового мира).
6. Инструкция по установке и использованию: Краткое описание того, как скопировать .jar файл в папку plugins, настроить config.yml и какие команды доступны пользователям.
7. Ссылка на исходный код проекта на github.

## Варианты
1. **Ежедневные задания**: несколько простых заданий, которые обновляются раз в сутки (срубить 64 дерева, убить 10 зомби). Логика: сброс по таймеру, хранение статистики игрока между сессиями.
2. **Путешественник**: исследовать определенные биомы. Логика: Отслеживание входа в новый биом и выдача награды за первый вход в каждый из списка.
3. **Рыбак-экстремал**: поймать редкую рыбу или сокровище во время рыбалки. Логика: проверка пойманного предмета и выдача награды.
4. **Коллекционер мобов**: убить каждого типа враждебных мобов определенное количество раз. Логика: Создание "словаря", где ключ — тип моба, значение — счетчик.
5. **Садовод**: вырастить определенное количество пшеницы, моркови и т.д. Логика: использование события для отслеживания естественного роста или роста от костной муки.
6. **Алхимик-любитель**: сварить определенные зелья. Логика: отслеживание события завершение варки зелья и выдача награды.
7. **Задание на время**: сделать квест, который нужно выполнить за 5 минут (например, добежать до точки и вернуться). Логика: таймер, проверка на провал (смерть, выход).
8. **Коробки с сюрпризами**: игрок покупает ключ, открывает сундук и получает случайный предмет из списка. Логика: Реализация рандома с разной вероятностью выпадения (шанс 1% на алмаз, 50% на булыжник).
9. **Система уровней**: за убийство мобов, добычу ресурсов игрок получает опыт. С новым уровнем открываются новые возможности. Логика: Накопление опыта, повышение уровня, изменение максимума здоровья (атрибуты игрока).
10. **Кровавая луна**: раз в N игровых ночей наступает "Кровавая луна" — мобы не горят на солнце, появляются в огромных количествах и видят игрока сквозь стены. Логика: Таймер, массовый спавн мобов с измененными атрибутами.
11. **Биржа вакансий**: табличка с текстом [Job: Miner]. Игрок, нажав на нее, получает задание добыть 50 камня. При сдаче (через команду) получает деньги (виртуальные). Логика: работа с атрибутами, контроль выполнения.
12. **Ломбард**: игрок может заложить ценный предмет и получить 50% его стоимости. Чтобы выкупить обратно, нужно вернуть сумму + проценты в течение часа. Логика: Хранение данных о закладе, таймер на потерю предмета.
