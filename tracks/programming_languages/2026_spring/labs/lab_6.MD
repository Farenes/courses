# Лабораторная №6. Разработка Bash-скрипта

## Тема и цель работы
Тема: Практическое освоение Bash для автоматизации рутинных задач системного администрирования, связанных с управлением файлами и каталогами.

Цель:
1. Закрепить навыки написания Bash-скриптов с использованием базовых конструкций (циклы, условия, функции).
2. Научиться применять стандартные утилиты командной строки в комбинации для решения практических задач.
3. Развить понимание принципов работы с файловой системой Linux (права доступа, метаданные, структура каталогов).
4. Получить опыт написания скриптов, которые можно использовать в реальной работе для упрощения рутинных операций.

## Общая постановка задачи
Разработать скрипт на языке Bash, без использования других языков. Допустимо применение Linux-утилит из стандартного пакета утилит.

## Предварительные требования
1. Операционная система Linux или MacOS или установленный WSL (Windows Subsystem Linux) или установленная виртуальная машина с любым Linux-дистрибутивом.

## Требования и рекомендации к скрипту
1. Скрипты должны начинаться с shebang (`#!/bin/bash`) и иметь комментарии, поясняющие ключевые моменты.
2. Код должен быть читаемым: используйте осмысленные имена переменных, разделяйте логические блоки.
3. Обязательна базовая обработка ошибок: проверка количества/корректности аргументов, существования файлов/директорий, проверка кодов возврата критичных команд (например, `tar`, `rm`).

## Отчет о выполнении
1. Титульный лист
2. Цель и задание работы
3. Полный код скрипта
4. Скриншоты демонстрации работы скрипта с изначальными условиями и результатом работы 
5. Код скрипта размещенный в публичном Github репозитории (ссылка в отчете)

## Варианты
1. **Организатор**. Напишите скрипт, который сортирует файлы в текущей директории по расширениям. Скрипт должен создавать папки (например, jpg_dir, txt_dir, pdf_dir) и перемещать в них соответствующие файлы.
2. **Датчик свежести**. Напишите скрипт, который находит все файлы в заданной директории (и ее поддиректориях), которые были изменены за последние 24 часа, и выводит их полные пути.
3. **Архиватор логов**. Напишите скрипт, который находит все файлы с расширением .log в заданной директории (или любой другой, доступной студенту), создает архив logs_backup_<дата>.tar.gz и копирует в него файлы старше 7 дней.
4. **Счетчик типов**. Напишите скрипт, который рекурсивно обходит указанную директорию и выводит количество файлов каждого типа (расширения). Вывод должен быть отсортирован по убыванию количества.
5. **Замена текста в файлах**. Напишите скрипт, который рекурсивно заменяет все вхождения строки old_string на new_string во всех файлах с расширением .txt в указанной директории. Используйте sed.
6. **Простой парсер CSV**. Напишите скрипт, который принимает CSV-файл (разделитель - запятая) и номер столбца N, а затем выводит все значения этого столбца.
7. **Монитор диска**. Напишите скрипт, который проверяет использование диска в указанной директории и выводит топ-10 самых больших файлов или папок. Используйте du и sort.
8. **Генератор отчетов**. Напишите скрипт, который анализирует текстовый файл, подсчитывает количество строк, слов и символов, и записывает эту статистику в новый файл report.txt.
9. **Генератор паролей**. Напишите скрипт, который генерирует случайный пароль заданной длины (по умолчанию 12 символов), состоящий из букв (верхний и нижний регистр), цифр и специальных символов. Скрипт должен принимать длину пароля как аргумент командной строки.
10. **Анализатор прав доступа**. Напишите скрипт, который рекурсивно проверяет файлы в указанной директории и находит все файлы, имеющие права доступа 777 (rwx для всех), и выводит их список.
